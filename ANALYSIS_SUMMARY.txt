DESIGN PRINCIPLES TO MAINTAIN
================================================================================

1. TYPE1 → DIFF → TYPE2 PIPELINE
   Never bypass diff; always audit-then-act
   Type2 only modifies what Type1 detected

2. MANDATORY DIFF DISCIPLINE
   Empty diff = Type2 skipped (no blind changes)
   Ensures safety and predictability

3. OS CONTEXT AWARENESS
   Check $global:OSContext.IsWindows11 for conditional logic
   Use OS-specific baseline sections

4. CORE FUNCTION SHARING
   All shared logic in Maintenance.psm1
   Type1 and Type2 modules only implement specific logic
   Prevents duplication and version mismatches

5. CONSISTENT MODULE PATTERN
   Type1: Load baseline → Compare → Save diff → Return result
   Type2: Get diff → Process items → Return result
   Follow this religiously for new modules

6. REBOOT FLAG DISCIPLINE
   Only Type2 modules set RebootRequired in result
   Stage 5 checks all results to decide if system reboot needed
   Reboot detection at startup for Windows Update restart loops

7. LOGGING FOR DIAGNOSTICS
   Write-Log with component tags for filtering
   Use DEBUG level for verbose details
   Transcript captures everything for HTML report
