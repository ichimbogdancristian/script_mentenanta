The script should become aware of its environment, including the username and path.
In the same directory, the script should create maintenance.log and log all actions.
It should check if it has admin rights; if not, it should relaunch itself with admin rights.
The script should check if System Restore is enabled on drive C. If not, it should enable it and create a restore point.
The script should check if there is a scheduled task to launch itself on the first day of every month at 1am.
If not, it should create this scheduled task.
Next, check if there is a startup task to launch the script at user login with a 1-minute delay. If there is, remove it.
Check if the system has any pending restarts.
If there are pending restarts, the script should create a startup task to launch itself at user login with a 1-minute delay.
Afterward, it should restart the system so that the pending changes take effect.

The script should check if WinGet (Microsoft.AppInstaller) is available.
If not, it should download and install it from the internet.
Once WinGet is available, it should be used to install PowerShell 7 (Microsoft.PowerShell).
Next, check one by one if each of the following dependencies is available:
VBScript (cscript.exe)
Windows Task Scheduler (schtasks.exe)
DISM
PSWindowsUpdate
Appx
NuGet CLI (nuget.exe) and NuGet Provider
If any are not available, the script should install them unattended.
Next, download the repository to the same path as script.bat and extract it.
From the repo folder, execute script.ps1 in a PowerShell 7 environment with admin rights.
After launching script.ps1 in a new PowerShell 7 window, display a 20-second countdown and close the window, with the option to abort the countdown and prevent closing.
Every operation in this script should proceed unattended.
